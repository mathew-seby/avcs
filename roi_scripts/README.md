# AVCS: Automated ROI Selection and Dataset Annotation Toolkit

This toolkit contains the complete preprocessing and auto-labeling pipeline for the nighttime AI-driven Traffic Control System. These scripts work together to define specific traffic lanes, monitor vehicle density to prevent lane starvation, and automatically generate YOLO training labels for the dataset.

## 🗂️ File Descriptions & Screenshots

### 1. `get_roi.py` (Interactive Lane Selector)
A Python script that provides a visual Matplotlib interface to manually define a target lane on a sample image. 
* **How it works:** It loads a traffic image and allows the user to click four points in clockwise order (Top-Left, Top-Right, Bottom-Right, Bottom-Left) to draw a custom polygon over the specific lane.
* **Output:** It calculates the normalized coordinates (0.0 to 1.0) for the polygon and prints them directly to the terminal for easy copying. 

### 2. `yolov8_format.py` (Coordinate Converter)
Because YOLO models require standardized bounding boxes rather than irregular polygons, this helper script bridges the gap.
* **How it works:** It takes the four normalized polygon coordinates generated by `get_roi.py` and mathematically converts them into the standard YOLO bounding box format (`x_center`, `y_center`, `width`, `height`).
* **Output:** A single formatted string of four values ready to be plugged into the main annotation pipeline.
